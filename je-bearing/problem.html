Operating Context

Why this problem matters

A high-mix CNC job shop is one of the hardest environments to schedule. Understanding why requires understanding the operating model — and why it breaks in exactly predictable ways.

The Operating Model

What makes a HMLV shop different

🏭

High-Mix, Low-Volume

Dozens of different part families, typically 20–200 units per run. No two weeks look the same. Standard scheduling heuristics built for high-volume production fail here.

👷

Operator Coverage Model

Each skilled operator covers 2–3 machines simultaneously. This means one operator's attention is divided — and divided attention creates elastic, unpredictable actual cycle times.

📋

Static ERP Schedules

ERP systems record planned times against fixed routings. Reality diverges by day 2. Setups run long. Operators hop. The schedule becomes reference documentation, not operational truth.

Root Cause Analysis

Why static scheduling breaks in this environment

1. Elastic Cycle Times

A CNC mill running unattended has a deterministic cycle time. But the effective cycle time when an operator monitors 3 machines is variable. When machine A needs attention, machines B and C wait.

Effective cycle = nominal cycle × (1 + attention_dilution × (machines_covered - 1))
At 3 machines: +30% average cycle elongation

2. Setup Drift

Setups for new part families are inherently variable. Fixturing adjustments, first-piece inspection, program edits — all invisible to a static schedule that books 1.5h and moves on.

Setup completed on time65% of the time

3. The Batching Dilemma

Every setup has a fixed cost. Running more parts amortizes that cost — but builds inventory. The optimal batch size is a function of carrying cost, demand timing, and operator availability. ERP doesn't optimize this.

4. The Weekly Meeting as Optimization Engine

The Monday planning meeting is where the real scheduling actually happens. An experienced planner mentally models:

Priority Which customers must ship this week, non-negotiable
Capacity Who can cover what, given certifications and load
Setups Where can we consolidate to save changeover time
Risk What might go wrong, and what's the fallback

This intelligence is real. It's just entirely tribal and person-dependent.

The Binding Constraint

Skilled attention is the real bottleneck

Machine capacity is abundant in most HMLV shops. The constraint is the skilled operator who knows each machine, each part family, each customer's expectations — and who can only be in one place at a time.

Machines the average operator is certified on. Machine assignment is not free — it's bounded by skill and trust.

Why the right answer is decision support, not automation

❌

Full Automation

Ignores the tacit knowledge operators carry. An algorithm that doesn't know Joe prefers VMC1 because it runs tighter tolerances will make systematically wrong assignments.

⚠️

Pure Manual

The current state. Effective when the right person is in the room. Fails on vacations, transitions, and when the number of variables exceeds working memory.

✅

Decision Support

Surface the trade-offs. Encode the logic. Let humans decide — but faster, with more information, and with the implications visible before the decision is made.
